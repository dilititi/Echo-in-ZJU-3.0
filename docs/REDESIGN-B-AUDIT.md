## 3. **更好的方案：极简界面 + 渐进披露（强烈推荐）**

> 你已经主动提了这个方向——"收纳在网页的某个开关后面"。下面是完整设计。

### 3.1 一个 ⋯ 菜单托管所有"次级"功能

将 Header 右上角的 ☰ **改为单个 ⋯ 图标**，点击展开一个右侧抽屉（drawer，宽 ~320px，纸本风）。抽屉里按重要性分组：

```
⋯ More
├─ 模式
│  ├─ 🏙️ 3D 校园
│  ├─ 🎧 声音探索（听音辨物）
│  └─ 🗺️ 图层（默认 / 夜间 / 卫星）
├─ 创作
│  ├─ 📁 上传音频谜题
│  ├─ ✨ 创建自定义事件
│  └─ 🎵 图书馆挑战
├─ 社交
│  ├─ 💬 声音社区（墙 + 榜）
│  └─ 👤 昵称设置
├─ 设置
│  ├─ 🌙 主题（日 / 夜 / 纸本）
│  ├─ 📖 帮助 & 引导
│  └─ 🗑️ 重置所有数据
└─ 状态
   └─ ● 后端在线 · OSS 模式
```

**收益：**
- Header 只剩"品牌 + stats + ⋯"三块，与 reference 一致
- **0 功能损失**
- 用户高频路径（录音、看 Field Notes）零点击触达；低频路径（设置、社区）多 1 次点击

### 3.2 Field Notes 卡片承担三重身份

- 列表项 = 我的录音 + 默认谜题 + 自定义事件 + 收集到的声音
- 用 `note-stamp` 的图标区分类型：
  - `🎤` 我的录音
  - `🔒` 谜题（未解锁）
  - `✓` 谜题（已解锁）
  - `📌` 自定义事件
  - `♬` 关卡音频
- 列表按时间倒序排，**天然就是声音游记**
- 顶部右上加一个微型 filter chip 行（"全部 / 我的 / 谜题 / 事件"），可省略

### 3.3 关卡系统融入 Header stats

```
关卡 03 · 录音 24 · 解锁 17/30
└──────┬───────┘
       └─ 点击 → 弹 levelInfo 详情（沿用 levelBadge 现有交互）
```

进度条改为 stats 下方一根 1px 细线，宽度按完成度填充。**信息密度高，占位极小。**

### 3.4 Marker 类型扩到 6 类对齐 reference

现状 5 类（sport/club/memory/love/study）→ 加 `sound` 凑齐 6 格的 3×2 网格。Sound 类可代表"声景标记"，与现有 sound-explore 解锁打通。

### 3.5 录音体验保留在主面板

Field Notes 卡片底部固定：

```
●  [给声音起个名字…              ]
   [地点标注 · 简短描述           ]
```

录音中 `●` 变 `⏸`，下方多一行波形/计时（用 `recordingStatus` 现有 hook）。**与 reference 完全一致，不损失任何功能。**

## 6. 已确认的决策（2026-05-16）

1. **第 6 类 marker `sound`** = **用户上传的音频**。Markers 卡片 3×2 网格的 6 类锁定为：`study / love / sport / club / memory / sound`。选择 `sound` 后放置的标记关联到用户上传音频列表。
2. **3D 模式**：保留入口，**降级为"图层"中的一项**。⋯ 抽屉的 `图层` 区合并显示：默认 / 夜间 / 卫星 / **3D 校园**。点击 3D 校园仍走 `switchTo3D()` 跳转。
3. **主题**：**砍掉日 / 夜切换**，UI 锁定纸本日间。`themeToggleBtn` 和 `data-theme` / `THEME_KEY` 全部移除。夜间主题留作 3D 模式未来开发项。
4. **闪烁地标提示** `recommendHint`：废除浮窗，改为 **Field Notes 卡片顶部的一行 hint 文案**（首次访问显示，点击关闭后 localStorage 记住）。
5. **自定义事件创建**：接受**移入模态**（⋯ 抽屉 → 创作 → ✨ 创建事件）。
