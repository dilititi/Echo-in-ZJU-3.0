# 交互式校园声音地图

一个基于 Leaflet.js 的交互式校园地图应用，支持地图标记、音频录制、声音谜题和关卡式学习。

## 功能特点

### 核心功能
- **地图交互**：使用 Leaflet.js 显示校园地图，支持缩放、拖拽
- **标记系统**：在地图上添加各种设施图标（教室、图书馆、餐厅、体育设施）
- **音频录制**：支持浏览器录音，自动上传到服务器
- **音频上传**：支持上传本地音频文件（支持 .wav, .mp3, .ogg, .m4a）
- **音频播放**：点击标记播放关联的音频，带进度条显示
- **声音谜题**：预设音频谜题，通过匹配解锁校园故事

### 学习系统
- **6个关卡**：从认识校园到自由探索，循序渐进
- **功能解锁**：每关完成后解锁新功能
- **新手引导**：11步交互式引导，帮助用户快速上手

### 数据管理
- **本地存储**：用户进度、标记、音频笔记保存在浏览器 localStorage
- **后端支持**：Flask 后端服务，支持本地文件存储和阿里云 OSS
- **音频池**：音频对象复用，提升播放性能
- **列表缓存**：音频列表缓存机制，减少服务器请求

## 技术架构

### 前端模块结构
```
js/
├── config.js      # 配置模块（服务器地址、限制参数等）
├── data.js        # 数据模块（谜题、关卡、建筑、表情）
├── utils.js       # 工具模块（验证、转义、防抖）
├── storage.js     # 存储模块（localStorage 操作）
└── app.js         # 主应用模块（核心逻辑）
```

### 后端架构
- **server.py**：Flask API 服务
- **storage.py**：存储管理器（本地/OSS 双模式）
- **config.py**：配置管理（环境变量加载）
- **utils.py**：工具函数（限流、认证、缓存）

## 快速开始

### 环境要求
- Python 3.7+
- 现代浏览器（支持 MediaRecorder API）

### 安装依赖
```bash
pip install -r requirements.txt
```

### 启动服务
```bash
python server.py
```

服务将在 `http://127.0.0.1:8081` 启动。
26/4以上载到https://echo-zju-3-0.onrender.com/

### 访问应用
使用浏览器打开 `index.html`，或使用任意 Web 服务器：
```bash
python -m http.server 8000
```
然后访问 `http://localhost:8000`

## 使用说明

### 基本操作
1. **选择图标**：点击左侧控制面板的图标
2. **放置标记**：将鼠标移到目标位置，按 **N键**
3. **录制音频**：在右侧面板输入名称，点击"开始录音"
4. **选择音频**：点击音频列表的"选择"按钮
5. **播放音频**：点击地图上的标记
6. **删除标记**：点击标记选中，按 **Backspace键**

### 关卡说明
| 关卡 | 标题 | 解锁功能 |
|------|------|----------|
| 1 | 认识校园 | 地图拖拽、缩放 |
| 2 | 放置标记 | 添加标记、选择图标 |
| 3 | 录制声音 | 录音、上传音频 |
| 4 | 绑定声音 | 绑定音频到标记 |
| 5 | 声音谜题 | 解锁系统谜题 |
| 6 | 自由探索 | 全部功能 |

## 配置说明

### 环境变量
创建 `.env` 文件配置：
```
OSS_ACCESS_KEY_ID=your_access_key
OSS_ACCESS_KEY_SECRET=your_secret_key
OSS_BUCKET_NAME=your_bucket
OSS_ENDPOINT=oss-cn-hangzhou.aliyuncs.com
API_KEY=your_api_key
DEBUG=False
```

### 本地存储
- `local_storage/audio/`：用户上传的音频
- `default_audio/`：预设谜题音频
- `icons/`：地图标记图标

## 性能优化

### 已实现的优化
1. **音频对象池**：复用 Audio 对象，减少 GC
2. **列表缓存**：30秒缓存音频列表
3. **事件清理**：播放结束清理事件监听器
4. **防抖机制**：配置了防抖等待时间
5. **一次性监听**：使用 `{ once: true }` 优化事件绑定

### 内测建议
- 关注音频列表加载速度
- 测试大量标记下的地图性能
- 验证音频播放的流畅性
